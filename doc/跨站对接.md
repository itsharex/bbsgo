# BBSGO 共享域 Cookie 单点登录方案

## 概述

BBSGO 支持通过共享域 Cookie 实现与主站的单点登录。用户在主站登录后，访问 BBSGO 时，BBSGO 中间件自动检测 Cookie 并完成登录。

---

## 名词解释

| 术语 | 说明 |
|------|------|
| 主站 | 已存在用户系统的网站，如 www.mageg.cn |
| BBSGO | 论坛系统，如 bbs.mageg.cn |
| 共享域 Cookie | domain 为 `.mageg.cn` 的 Cookie，会自动携带到所有子域名 |

---

## 对接流程图

```
┌─────────────────┐                      ┌─────────────────┐
│    主站         │                      │    BBSGO        │
│  www.mageg.cn   │                      │  bbs.mageg.cn   │
└────────┬────────┘                      └────────┬────────┘
         │                                         │
         │  1. 用户已登录主站，cookie 已写入        │
         │     （domain 为 .mageg.cn）              │
         │                                         │
         │  2. 用户访问 BBSGO 任意页面              │
         │  ─────────────────────────────────────►│
         │     （浏览器自动带上 .mageg.cn cookie）   │
         │                                         │
         │  3. SSO 中间件拦截所有 API 请求          │
         │     从 cookie 取 token                  │
         │     调主站 verify 接口验证               │
         │     POST /api/sso/verify                │
         │     Body: {"token":"xxx"}               │
         │  ◄────────────────────────────────────│
         │     返回: {uid, account, nickname,      │
         │           avatar}                       │
         │                                         │
         │  4. 查找/创建本地用户                   │
         │     生成 BBSGO token                    │
         │     写入 Set-Cookie: bbsgo_token=xxx   │
         │                                         │
         │  5. App.vue onMounted 检测到 cookie    │
         │     同步 token 到 localStorage          │
         │     调用 /user/profile 获取用户信息     │
         └─────────────────────────────────────────┘
```

## 工作原理

### 中间件自动处理

SSO 登录完全由后端中间件自动处理：

1. **SSO 中间件** (`middleware/sso.go`) 拦截所有 API 请求
2. 检查请求 Cookie 中是否有 `sso_cookie_name` 配置的字段
3. 如果存在，调用主站 `sso_verify_url` 验证
4. 验证通过后：
   - 查找或创建本地 BBSGO 用户
   - 生成 BBSGO JWT token
   - 将 token 写入 `Set-Cookie: bbsgo_token=xxx`
   - 在请求 header 中设置 `Authorization: Bearer xxx`

### 前端自动同步

前端 App.vue 在 `onMounted` 时检测 cookie 中是否有 `bbsgo_token`，有则同步到 localStorage 并调用 `/user/profile` 获取用户信息。

## 主站需要做的事

### 1. 确保 Cookie 设置共享域

用户登录后，设置 Cookie 时指定 `.yourdomain.com` 作为 domain：

```go
// 示例：Go 语言设置 Cookie
http.SetCookie(w, &http.Cookie{
    Name:     "token",
    Value:    userToken,
    Domain:   ".mageg.cn",  // 关键：设置为共享域
    Path:     "/",
    HttpOnly: true,
    Secure:   true,         // 生产环境建议 true
})
```

### 2. 提供验证接口

```go
// 处理 BBSGO 发来的验证请求
func HandleSSOSsoVerify(w http.ResponseWriter, r *http.Request) {
    var req struct {
        Token string `json:"token"`
    }
    json.NewDecoder(r.Body).Decode(&req)

    // 验证 token，获取用户信息
    user := validateToken(req.Token)

    avatarId := generateAvatarId(user.Uid)
    avatarURL := fmt.Sprintf("https://%s/api/avatar?code=%s", r.Host, avatarId)

    // 返回用户信息
    json.NewEncoder(w).Encode(map[string]interface{}{
        "uid":      user.Uid,
        "account":  user.Account,
        "nickname": user.Account,
        "avatar":   avatarURL,
    })
}
```

**请求**：
```json
POST /api/sso/verify
Content-Type: application/json

{
  "token": "主站颁发的 token"
}
```

**响应**：
```json
{
  "uid": 123,
  "account": "zhangsan",
  "nickname": "张三",
  "avatar": "https://www.mageg.cn/api/avatar?code=abcd"
}
```

**响应（token 无效）**：
```json
{
  "error": "invalid_token"
}
```

## BBSGO 端配置

### 1. 在管理后台添加配置

管理后台 -> 网站配置 -> 单点登录：

| 字段 | 说明 |
|------|------|
| Cookie 字段名 | 主站 cookie 中存放 token 的字段名（默认 token） |
| 验证接口地址 | 主站 verify API 地址 |

### 2. 配置示例

| key | value | 说明 |
|-----|-------|------|
| sso_cookie_name | token | 主站 cookie 中 token 的字段名 |
| sso_verify_url | https://www.mageg.cn/api/sso/verify | 主站验证接口地址 |

## 前端嵌入（一行代码）

在主站任意页面需要显示"BBS入口"的位置加入：

```html
<a href="https://bbs.mageg.cn/">进入BBS</a>
```

**不需要任何参数！** 用户访问 BBSGO 任意页面，SSO 中间件自动检测 Cookie 并登录。

## 安全考虑

1. **HTTPS**：生产环境务必使用 HTTPS
2. **Cookie 安全**：
   - 主站 Cookie 设置 `HttpOnly: true` 防止 XSS 读取
   - BBSGO Cookie 设置 `HttpOnly: false` 让前端 JS 可读取并同步到 localStorage
   - 设置 `Secure: true` 仅 HTTPS 传输
3. **Token 时效**：主站 Token 应有合理时效
